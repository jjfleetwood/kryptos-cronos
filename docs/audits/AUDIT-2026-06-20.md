## 1. Scope

This run covers the full default rotation in one pass:

1. **Code health** — oversized files, `console`/`any`/TODO debt, `dangerouslySetInnerHTML` sinks.
2. **Content integrity** — duplicate stage ids, empty briefings, quiz quality.
3. **Security & data** — a read of the forbidden zones (auth, webhooks, crypto, session, stage-flags, proxy) for secrets, XSS, SSRF, authz gaps, and flag exposure.
4. **Dependency / supply chain** — lockfile/dependency posture (with a coverage caveat, §3).
5. **Documentation / count integrity** — headline-count drift across the app + docs.

**Autonomy / blast radius.** The product is a Next.js 16 web app + API. It handles **authentication** (PBKDF2 + Supabase), **payments** (Stripe + RevenueCat webhooks), and **PII in Redis** (emails, progress, entitlements). The agent fleet that maintains it is *report-only* and gated by a global kill switch (`AGENTS_ENABLED`) + a bearer token (`AGENT_REPORT_TOKEN`) — the worst realistic blast radius of a wrong *agent* action is a spurious board card, not a code change or a mutated user record. The worst realistic blast radius of a platform defect in scope is account/entitlement compromise or PII exposure.

**Regulatory frame.** Consumer auth + card payments → the relevant control frames are **OWASP ASVS / Top 10**, **CWE**, and PCI-adjacent handling discipline for the Stripe path. AI-surface guidance (NIST AI RMF, OWASP-LLM, MITRE ATLAS) is in scope for the ARIA hint endpoint and the agent fleet, but this run surfaced no AI-surface findings.

### What was checked and is clean

- **Code health (E1):** 0 high/0 medium across 178 files. The single `dangerouslySetInnerHTML` (`apps/web/src/app/layout.tsx:97`) is a **static, nonce-guarded anti-FOUC script** with a constant `__html` — no user input, CSP-nonced. Not an XSS sink.
- **Typecheck (E6):** clean. **Lint (E5):** 0 errors (warnings only).
- **`proxy.ts`:** admin routes gated by `verifyAdminToken` at the edge; per-request nonce CSP with no `unsafe-inline` in `script-src`; `/api` CORS is **credential-less + origin-allowlisted**; `frame-ancestors 'none'`. Sound.
- **Session/crypto (`server-session.ts`, `crypto-utils.ts`):** HMAC-SHA256 session tokens with constant-time verify; PBKDF2-SHA256 **600k** iterations; CSPRNG salt. *Robustness note (not a finding):* a username containing `:` would break the `u:{user}:{hmac}` token parse — but I **verified by trace** it self-rejects (length mismatch → `null`), so it is **not** an impersonation/escalation vector. At most such an account can't hold a web session; a charset whitelist at registration (`/^[a-z0-9_-]+$/i`) would be a tidy hardening, not a security fix.
- **`stage-flags.ts`:** `import "server-only"`, **not** barrel-exported from `@kryptos/core`, imported by exactly two server routes (`check-flag`, `progress`). No client exposure. `check-flag` enforces auth + tier access + per-(user|IP)+stage rate limit (30/600s) + **constant-time** flag compare; XP scoring uses server-tracked signals only (client `timeTakenMs` drives only a self-penalty). Sound. *(Minor note: `check-flag` logs the correct flag value into the admin-only `admin:flag-log` Redis list — admin-gated telemetry; acceptable, flagged here for awareness only.)*
- **Webhooks:** Stripe verifies `constructEvent` signature before acting; RevenueCat verifies the shared-secret auth header via a length-independent constant-time HMAC compare; both downgrade tier only when **no** other entitlement source is active. Sound.
- **`admin-session` / `admin-token`:** admin cookie is minted only from a **session-verified** identity (never the request body — the prior takeover bypass is closed); v2 tokens are HMAC-signed with **8h** expiry; privileged routes layer Redis revocation via `requireAdmin`. Sound.
- **Identity resolver (`api-auth.ts` / `supabase-jwt.ts`):** bearer identity resolved from the **verified email claim → `email:{email}` index**, never spoofable `user_metadata`. Sound.
- **Secrets scan:** no hardcoded `sk_live`/`whsec_`/`sk_test_`/AWS keys/PEM/inline passwords in `apps/web/src` (E9 grep, 0 matches).
- **Count integrity (E3):** no in-app count drift; `check:meta` passes (build not blocked).
