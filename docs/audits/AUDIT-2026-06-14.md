## 3. Tests performed (control claims verified, not assumed)

- **Claim: `/studio` is admin-only.** *Tested.* The route is **not** in `proxy.ts`'s edge-admin guard —
  that guard only matches `pathname.startsWith("/admin")` (`apps/web/src/proxy.ts:37`). The studio page is
  a `"use client"` component whose entire manuscript is bundled and rendered; the only gate is a
  client-side `fetch("/api/auth/me")` and a render-block when `isAdmin === false`
  (`apps/web/src/app/studio/page.tsx:49–67`). The initial state is `isAdmin = null`, which falls through
  to the full-content `return` (lines 69+). **Result: FAIL — broken access control (MED-1).**
- **Claim: admin identity is not spoofable.** *Tested — PASS.* `/api/auth/me` derives `isAdmin` from
  `verifyAdminToken(admin_token cookie)` (HMAC), not from request body or `user_metadata`
  (`apps/web/src/app/api/auth/me/route.ts:8–9`). The gate's *logic* is sound; the gap (MED-1) is purely
  that confidential content is shipped client-side and rendered in the `null` window.
- **Claim: new Range CTF flags are server-only and don't leak.** *Tested — PASS.* `stage-flags.ts` still
  begins with `import "server-only";` (`packages/core/src/stage-flags.ts:1`). The `range-*.ts` content
  files contain only **open-ended fragments** (e.g. `FLAG{SQL1_` at `range-web.ts:128`, no closing brace)
  surfaced by terminal commands — the intended assemble-and-validate-server-side mechanic. No complete
  `FLAG{...}` appears in any client-reachable file, and **none** appear in `stages-meta.generated.ts`.
  `validate-ctf.mjs` confirms all 411 CTFs (incl. the new labs) assemble and validate.
- **Claim: the single `dangerouslySetInnerHTML` is safe.** *Tested — PASS.* It injects `antiFoucScript`,
  a **static template literal with no user interpolation**, on a `<script nonce={nonce}>` element
  (`apps/web/src/app/layout.tsx:50,97`). Not an XSS sink. (Recorded as LOW-1 for traceability only.)
- **Claim: headline counts are not drifting.** *Tested — PASS.* `drift-agent` reports **no count drift**
  at `stages: 871`; `docs-agent` reports `stale: 0, unstamped: 0` across 37 docs. (CLAUDE.md's narrative
  headline of 831 is stale prose but the in-app/locale numbers are reconciled, which is what drift gates.)

## 4. Findings

| ID | Severity | Class | Evidence | Framework | CAPA |

| MED-1 | **Medium** | Broken access control (confidentiality) | `proxy.ts:37` guards only `/admin`; `/studio` gate is client-only (`studio/page.tsx:49–67`), `isAdmin=null` renders full manuscript, and the manuscript ships in the client bundle | OWASP A01:2021 (Broken Access Control); CWE-602 (client-side enforcement of server-side security); CWE-200 (exposure) | **Engineer the control server-side**, do not patch the client. Preferred: render `/studio` as a server component that calls `requireAdmin(req)` and returns 401/redirect before emitting any manuscript HTML — OR add `/studio` to the `proxy.ts` edge-admin matcher so the route is blocked at the edge like `/admin/**`. Client `isAdmin` checks are the weakest control and must not be the only one. Owner: app/platform. Verify post-fix by requesting `/studio` with no `admin_token` and confirming no manuscript text in the response body or the route's JS chunk. |
| MED-2 | **Medium** | Content integrity (quiz quality) | `content-agent` aggregate: `quiz-length-bias=805` (longest-answer tell), `quiz-pool-under-10=860` (pools < 10 Qs) at `stages: 871` | OWASP-style assessment-integrity / item-writing best practice (no regulatory clause) | Administrative + engineering: continue the quiz-quality sprint — normalize answer lengths to kill the longest-answer tell and grow thin pools toward ≥10 items. This is a **pre-existing, tracked backlog** item (content-author / content-reviewer agents own it), not a regression from this sprint. No new exposure; do not block release on it. |
