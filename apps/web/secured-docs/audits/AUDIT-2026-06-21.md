## 1. Scope

This run covers the full default rotation in one pass, with depth concentrated on the security-and-data leg (where the real blast radius lives):

1. **Code health** — oversized files, `console`/`any`/TODO debt, `dangerouslySetInnerHTML` sinks.
2. **Content integrity** — duplicate stage ids, empty briefings, quiz quality.
3. **Security & data** — a read of the forbidden zones (auth, webhooks, crypto, session, stage-flags, proxy) plus the AI/LLM surface (`/api/hint`), for secrets, XSS, SSRF, authz gaps, prompt-injection, and flag exposure.
4. **Dependency / supply chain** — lockfile/dependency posture (with a coverage caveat, §3).
5. **Documentation / count integrity** — headline-count drift across the app + docs.

**Autonomy / blast radius.** The product is a Next.js 16 web app + API. It handles **authentication** (PBKDF2 + Supabase), **payments** (Stripe + RevenueCat webhooks), and **PII in Redis** (emails, progress, entitlements). The agent fleet that maintains it — including this Audit Agent — is *report-only* and gated by a global kill switch (`AGENTS_ENABLED`) + a bearer token (`AGENT_REPORT_TOKEN`); the worst realistic blast radius of a wrong *agent* action is a spurious board card, not a code change or a mutated user record. The worst realistic blast radius of a platform defect in scope is account/entitlement compromise or PII exposure.

**Regulatory frame.** Consumer auth + card payments → the relevant control frames are **OWASP ASVS / Top 10**, **CWE**, and PCI-adjacent handling discipline for the Stripe path (no card data is handled directly — Stripe-hosted checkout). The AI surface (ARIA `/api/hint`, the agent fleet) is in scope for **NIST AI RMF / OWASP-LLM / MITRE ATLAS**; it is **not** EU AI Act high-risk (training/education, advisory-only).
