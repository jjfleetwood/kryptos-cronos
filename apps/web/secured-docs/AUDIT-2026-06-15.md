## 1. Scope

**This run audits: Security & Data + Documentation/Governance integrity**, with code-health and content/drift swept as standing baselines.

Per the default rotation (*code health → content integrity → security & data → dependency/supply chain*), this engagement focuses the depth on the **security & data** leg — the auth/session/webhook/crypto/flag zones — because that is the highest-blast-radius surface and several sensitive-zone files were touched in the last 14 days (the Stripe webhook, the docs route allowlist). The deterministic code-health/content/drift agents were run as standing baselines. The **dependency/supply-chain** leg was attempted but `npm audit` could not complete in this environment (see §3.7) and is carried to the next run.

**System under audit.** Kryptós CronOS — a gamified cybersecurity/AI training platform (Next.js 16 / React 19 / Upstash Redis / Resend / Stripe / Supabase). It handles **authentication** (PBKDF2 + Supabase JWT), **payments** (Stripe + RevenueCat IAP), and **PII in Redis** (user records, email index, progress).

**Autonomy / blast radius.** The platform itself is not an autonomous agent, but it *hosts* a fleet of report-only dev agents that file findings to the Development board via `POST /api/agent/report`. That endpoint is the relevant autonomy surface:
- **Blast radius of the agent surface:** board cards only — agents are report-only, OWNER-scoped (each agent touches only `initiator = agent:<name>` cards), gated by a bearer token *and* a global kill switch (`AGENTS_ENABLED`). Worst realistic outcome of a compromise: board spam, not data/payment impact.
- **Blast radius of the product surface:** a session/auth or webhook-verification defect is the real risk — account takeover or tier/entitlement forgery. That is where this run spent its depth.

**Regulatory frame.** Not EU AI Act high-risk (training/education, advisory). Relevant frames: **OWASP ASVS / Top 10** (auth, session, access control), **PCI-adjacent** discipline around the Stripe path (no card data is handled directly — Stripe-hosted checkout), **CWE** for the concrete defects, and **NIST AI RMF (Govern/Manage)** for the agent-fleet controls.

## 2. Methodology

1. Ran the four deterministic fleet agents **dry** (no `--report`) from `apps/web` (they resolve `ROOT = cwd/../..`, so they must run from the workspace, not the repo root).
2. Ran `npm run lint -w @kryptos/web` and `npx tsc -p apps/web/tsconfig.json` as the build-gate baselines.
3. Attempted `npm audit` (dependency/supply-chain) — blocked in this environment.
4. Performed a manual **security read** of the forbidden/sensitive zones (auth, session, webhooks, crypto, stage-flags, proxy, flag/answer validation), assisted by a read-only sub-agent; every claim cites file:line.
5. For each control *claim* in the project's stated security posture, confirmed it actually holds against the code rather than assuming it.

All findings are **report-only**. No files in the forbidden zones (auth, payments, crypto, session, `stage-flags.ts`, `proxy.ts`) were edited.

### 3.8 Security read of forbidden/sensitive zones (manual + read-only sub-agent)

**Controls CONFIRMED holding (with evidence):**
- **Stripe webhook signature verified** — `apps/web/src/app/api/webhooks/stripe/route.ts:18` calls `stripe.webhooks.constructEvent(rawBody, sig, secret)` on the raw `req.text()` body and 400s on failure. Correct (no forged-event tier grant).
- **RevenueCat webhook auth is constant-time** — `apps/web/src/app/api/webhooks/revenuecat/route.ts:28-43` HMACs both sides before `timingSafeEqual`, length-independent, 401 on mismatch.
- **CTF flags are server-only** — `packages/core/src/stage-flags.ts:1` has `import "server-only"`; imported only by `check-flag/route.ts` and `progress/route.ts`; not barrel-exported (package `exports` is `./*`), not imported by any client component or by `stages-meta`. Flags are never client-reachable.
- **Flags never leak in responses** — `check-flag/route.ts` returns only `{ correct }`; the submitted flag is written solely to the admin-only `admin:flag-log` Redis list, not the HTTP body. `progress` POST refuses any flag-bearing stage (`progress/route.ts:60-65`), so completion cannot be forged by posting a stageId.
- **Constant-time flag compare** — `check-flag/route.ts:14-18` length-checks then `timingSafeEqual`.
- **Quiz answers spoof-safe** — `check-answer/route.ts` validates by option *text* server-side, never exposes `correctIndex`.
- **Rate limits present** — check-flag 30/600s per user+stage; check-answer 60/600s; leaderboard 30/min/IP. Identity derived from the verified token, never the request body (`progress/route.ts:39`).
- **PBKDF2-SHA256, 600k iterations** — `apps/web/src/lib/crypto-utils.ts:3,17` (OWASP-current).
- **Supabase JWT identity non-spoofable** — resolved from the verified `email` claim → `email:{email}` index, not `user_metadata` (`apps/web/src/lib/supabase-jwt.ts:38,62`).
- **Admin token v2** — HMAC + 8h expiry + timing-safe compare; legacy non-expiring tokens rejected (`apps/web/src/lib/admin-token.ts`).
- **Docs route hardened** — `apps/web/src/app/api/docs/[file]/route.ts:47` requires admin, and `:52` gates on an allowlist `Set` membership check (no path traversal possible — `..`/`/` aren't allowlisted), reading from a fixed `secured-docs` dir.
- **Agent report endpoint least-privilege** — `apps/web/src/app/api/agent/report/route.ts:27-34`: global kill switch (`AGENTS_ENABLED !== "true"` → 503) *then* bearer-token check; each agent dedups/auto-resolves only its own `initiator`-scoped cards.
- **No live hardcoded secrets** — all `sk_live`/`whsec_`/`AKIA`/`BEGIN…PRIVATE KEY`/`sk-ant-` hits are `.env.example` templates, CI placeholders (`whsec_ci-placeholder`), docs, or deliberate curriculum/CTF example strings.
- **No SSRF surface** — every server-side `fetch()` targets a hardcoded host (`api.anthropic.com`, `api.resend.com`); no user-controlled URL reaches a fetch.
- **The one `dangerouslySetInnerHTML`** (`apps/web/src/app/layout.tsx:97`) injects a static module-constant anti-FOUC script with no user input, served under the per-request CSP nonce — not XSS.
